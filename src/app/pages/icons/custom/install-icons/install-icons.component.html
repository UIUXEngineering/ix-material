Custom SVG icons are contained in their own project. Install the project as you would any other module.

It is recommended not to save the icons module in installing as the module is quite large, and can slow down your CI builds.

For development convenience, a CLI is created to concat svg icons into one file, namespacing them so it's easy to add to your project. Install the CLI globally:

Place a config file named in the _root directory of your project. The CLI will read this config to concat your custom icons.

  • svg ( ): svg config for cli processing.
    • tsReference ( ): TS file referencing final output of concatenated svg file. Used for Material loading so you don't have make many changes in your project if changing the name of the assets file or version.
    • sets ( ): Contains configs to create multiple concatenated svg files ( if you wanted multiple files).
      • version ( ): Adds version of output file. For example the 'v2' in
      • outDir ( ): Location to save concatenated svg file. For example
      • setName ( ): Location to save concatenated svg file. For example the 'icon-set' in
      • srcFiles ( ): Source svg file paths and names. Each key of this object is the path to a base directory to svg files.
        • key of object [path to an svg directory] ( ): Each key of this object is the path to a base directory to svg files.
          • key of object: [svg file name] ( ):
            • id ( ): Angular Material Reference name of svg file.
