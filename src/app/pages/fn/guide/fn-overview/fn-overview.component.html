The FN consists of utility functions and angular services to solve common problems.

In the ideal development scenario you get well structured data from an API, and display it in the UI with no iteration on any data set. And then you add data in the same fashion. But, in most cases as reality hits the road, you start working with complex architectures:

  • Legacy API's that are in maintenance mode.
  • No API development support.
  • Data sets that are not precisely as you need them.
  • Mixing new API's with legacy as they migrated.
  • Multiple API architectures -- GraphQL, REST, WebSockets
  • Etc., etc., etc.

When you spend significant time solving this problems, it would be nice to do a little obfuscation, unit testing, documentation, and then share rather than keeping code locked in multiple repos and relying on copy and paste to reuse.

And so here we are.

Care has been taken to keep a similar architecture to Angular FN. So you may install and use in the same manner.

The modules are just functions or classes unless documented otherwise. The FN platform is broken up into smaller modules for organization and tree shaking.
