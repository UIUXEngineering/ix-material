Proof of Concepts ( PoC ) are a way to flesh out concepts for anything -- algorithms, material based components, data services, etc. It's a playground to brainstorm.

There is a specific place to add PoCs ( per instructions below ) that allow fast testing and development. The reason is that if you develop directly in a platform project, you have to build to test and create demos ( time consuming ). PoCs are built directly into the doc app allowing more rapid developing. There is no risk merging to master as PoCs are not published to NPM. But they will be published to the doc sever for peer feedback. Hence code reviews may be used for brain storming just like standards checking.

Once a PoC is ready to be published, it may be migrated to it's appropriate place in the platform.
